# 技术架构说明

## 概述

本系统是一个**轻量级Agent**，采用**纯Python + 设计模式**实现，不依赖任何Agent框架或LLM。

## 🏗️ 架构设计

### 核心理念

> **"Agent"不一定需要LLM，关键在于自主性和智能决策**

本系统通过以下方式实现Agent特性：

```
┌─────────────────────────────────────────┐
│          Agent智能系统                   │
├─────────────────────────────────────────┤
│                                         │
│  感知层: 文件扫描 + 元数据提取          │
│  ↓                                      │
│  决策层: 规则引擎 + 模式匹配            │
│  ↓                                      │
│  执行层: 模块化转换器                   │
│  ↓                                      │
│  验证层: 质量检查 + 回归测试            │
│                                         │
└─────────────────────────────────────────┘
```

## 🛠️ 技术栈

### 核心依赖

| 库名 | 版本 | 用途 |
|------|------|------|
| `numpy` | >=1.21.0 | 数组处理、点云数据 |
| `opencv-python` | >=4.5.0 | 图像读取和处理 |
| `Pillow` | >=9.0.0 | 图像格式转换 |
| `PyYAML` | >=6.0 | 配置文件生成 |
| `open3d` | >=0.15.0 | 3D点云处理 |
| `pycocotools` | >=2.0.4 | COCO格式工具 |

### 无需依赖

- ❌ **LLM API** - 无需OpenAI、Claude等
- ❌ **Agent框架** - 无需LangChain、CrewAI等
- ❌ **网络请求** - 完全离线运行
- ❌ **数据库** - 纯文件系统操作

## 🧠 Agent能力实现细节

### 1. 感知能力（Perception）

**实现方式**: 启发式算法

```python
# src/inspect_data.py

class DataInspector:
    def _infer_dataset_type(self, file_counts):
        """通过文件扩展名统计推断数据类型"""
        has_images = file_counts.get('2d_image', 0) > 0
        has_pointclouds = file_counts.get('3d_pointcloud', 0) > 0
        has_temporal = self._check_temporal_structure()
        
        if has_temporal:
            return '4d_sequence'
        elif has_pointclouds:
            return '3d_pointcloud'
        else:
            return '2d_image'
```

**特点**:
- ✅ 基于文件扩展名识别（`.jpg`, `.pcd`, `.bin`）
- ✅ 通过目录结构推断（时间戳命名 → 4D序列）
- ✅ 采样策略减少IO开销

### 2. 决策能力（Reasoning）

**实现方式**: 规则引擎

```python
# src/format_converter.py

class FormatConverter:
    def convert(self):
        # 自动决策使用哪个转换器
        if data_type == '2d':
            return self._convert_2d()  # → COCO
        elif data_type == '3d':
            return self._convert_3d()  # → KITTI
        elif data_type == '4d':
            return self._convert_4d()  # → Waymo
```

**特点**:
- ✅ 基于检测结果自动路由
- ✅ 预定义的转换策略
- ✅ 可扩展的转换器架构

### 3. 执行能力（Action）

**实现方式**: 模块化设计

```python
# 转换器模块结构
converters/
├── coco_converter.py      # 2D → COCO
├── kitti_converter.py     # 3D → KITTI
└── waymo_converter.py     # 4D → Waymo
```

**特点**:
- ✅ 每种格式独立实现
- ✅ 统一的接口设计
- ✅ 易于添加新格式

### 4. 验证能力（Feedback）

**实现方式**: 规则检查 + 样本测试

```python
# src/validator.py

class DataValidator:
    def validate(self):
        self._validate_directory_structure()  # 目录检查
        self._validate_config_files()         # 配置检查
        self._test_sample_loading()           # 加载测试
```

**特点**:
- ✅ 多层次验证
- ✅ 实际加载测试
- ✅ 详细的错误报告

## 📊 与Agent框架的对比

## 🎯 设计哲学

### 为什么选择轻量级Agent？

1. **确定性任务**: 数据格式转换是确定性任务，不需要LLM的不确定性
2. **性能优先**: 处理大规模数据集时，速度至关重要
3. **离线运行**: 避免网络依赖和API成本
4. **可调试性**: 纯代码逻辑易于调试和维护

### 适合使用本架构的场景

✅ 结构化数据处理  
✅ 规则明确的任务  
✅ 需要高性能  
✅ 离线环境  
✅ 成本敏感

### 不适合的场景

❌ 需要理解自然语言  
❌ 处理完全未知格式  
❌ 需要创造性推理  
❌ 复杂的多步对话

```python
    tools=[

    llm=OpenAI()

```
